基于Linux系统的云服务器hexo博客远程搭建以及本地端服务的配置

一、云服务器的购买

  • 服务器我用的是阿里云的2核2G云服务器ESC，系统是Linux Ubuntu20.04 64位，先用这个尝试搭建个人博客。但其实腾讯和阿里的服务器差不了多少，有学生优惠的话都很便宜。
  • 域名可以去阿里云购买自己喜欢的，国内的域名是要备案的，大概是6-20个工作日，如果不想等的同学可以购买国外的域名。

二、关于云服务器的操作

1. 配置安全组

开阿里云服务器管理控制台->点击左侧安全组->点击右侧的配置规则->点击添加安全组规则

2.服务器环境搭建

  • 1. 我们需要nginx作为我们的服务器，所以我们首先要安装nginx。

    Ubuntu安装命令如下：
    sudo apt update
    sudo apt install nginx
    Centos安装命令如下：
    yum install -y nginx
    安装完成后，我们启动服务器：
    systemctl start nginx
    systemctl enable nginx
    其中/etc/nginx/是nginx默认的配置路径，等下会用到。

  • 2.配置服务器路由

    安装并启动服务器后，我们就完成了第一步，现在我们可以尝试使用自己的电脑去访问服务器的公网IP。可以发现，公网IP可以打开一个nginx的默认网页。这样，我们就成功了第一步。

    但是实际上是想要让这个地址指向我们的博客，而不是nginx的默认网址，这就需要我们去配置nginx的配置文件。大部分关于centOS的资料都是说配置位于etc/nginx/conf.d/ 下的dafault.conf 这个配置文件，但是我在配置的时候并没有找到它，后来查看了一下文件结构，阿里云默认的库下载的是fedora版本的nginx，所以我们应该配置的是位于 etc/nginx/ 下的 nginx.conf 。

    我们先不急着打开这个文件，因为我并不建议直接配置这个文件，我们应该先创建一个新的文件，然后采用include的方式，将这个文件包含进nginx.conf中。
    操作如下：
    在/etc/nginx/目录下创建一个文件夹 叫 vhost

    编辑 blog.conf文件内容

    保存并退出:wq
    打开/etc/nginx/目录下的nginx.conf文件
    vi /etc/nginx/nginx.conf

    也可以通过Xftp对服务器进行文件管理，可以很方便的打开文件并编辑
    如果以后还想添加新的网站，也可以在vhost目录下新建一个conf配置文件。

  • 3.安装Git以及Node.js
    • 1.安装Node.js
      Centos： Ubuntu: 安装完成后执行 node -v 和 npm -v 如果打印版本号则安装成功
    • 2.安装Git及配置仓库
      这一部分主要目的是让我们个人的电脑可以通过ssh方式连接到云服务器，然后我们就可以通过命令行方式将我们的博客Po到服务器上。操作如下：
      安装git：
      Centos：
      yum install git
      Ubuntu:
      apt install git
      配置git用户
      adduser git
      修改用户权限 找到这个位置添加下面这句话

      保存退出后 将sudoers文件权限改回原样
      chmod 400 /etc/sudoers
      设置git用户的密码
      sudo passwd git
      切换到git用户，然后在~目录下创建.ssh文件夹
    • 生成公钥密钥文件*
      ssh-keygen
      此时在目录下就会有两个文件，分别是
      id_rsa 和 id_rsa.pub
      其中 id_rsa.pub 就是公钥文件 我们复制一份
      cp id_rsa.pub authorized_keys
      这样目录下就会有一个authorized_keys文件，它和id_rsa.pub一模一样。
      最后我们修改它的权限 然后我们在自己的电脑上,注意，是自己的电脑上
      打开cmd，使用ssh方式连接我们的云服务器。

      最后提示
      Welcome to Alibaba Cloud Elastic Compute Service !
      说明登录成功了。
      现在我们要创建一个git的仓库，并且新建一个post-receive文件，操作如下 输入以下内容：
      git --work-tree=/home/www/website --git-dir=/home/git/blog.git checkout -f
      保存退出并授予该文件可执行权限
      chmod +x ~/blog.git/hooks/post-receive
      至此我们就完成了所有关于服务器端的配置.

四、完成部署

完成部署后可以去Hexo主题下载自己喜欢的主题，美化博客。现在去本地主机浏览器上输入域名或者公网IP，访问你的博客吧！

五、部署过程中踩到的坑及其解决办法

  • 在执行最后一步时hexo clean && hexo generate --deploy出现报错如下

    这是因为创建本地仓库时未定义用户名和邮箱，此时再按照错误提示，利用 然后再执行命令就可以了
  • 在将文件push到远程服务器上之后访问显示403forbidden，然后再找了找自己服务器上的目标文件夹./home/www/website发现该目录下没有任何的文件，说明push失败了，原因是因为因为/home/www/website没有权限，设置一下权限就好了,在/home目录下 使用chmod -R 777 ./www
    最后再nginx -s reload发现成功将自己的hexo博客部署至远程服务器。
